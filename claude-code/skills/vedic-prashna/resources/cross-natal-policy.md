# 可选本命交叉 · 判读消费入口（默认关，矛盾裁决专用）

> **本文件落实 SKILL.md §可选本命交叉的 6 条硬约束**，给出 V1 具体判据规则(仅"矛盾裁决"一条)与未来产品扩展的加规则规范。
> **定位**：Prashna 主判读**自足**——Parashari + 提问盘足够独立回答"能不能/何时/吉凶"。本命交叉**不作辅证增强**、**不作锦上添花**，只作"矛盾裁决"通道。
> **传统依据**：BPHS(Parashara)——本命=根盘、Prashna=事件盘；事件不能违反根盘大方向。KN Rao——Prashna 严重打架时以本命为准。

---

## 1. 触发条件

用户显式说以下任一 → build 时加 `--cross-natal <本命 structured_data.md 路径>`：

- "结合我的本命看这个 Prashna"
- "用我今天的 Dasha 看看这个卦"
- "我的本命 Dasha 跟这个 Prashna 结论对得上吗?"
- "本命裁决一下"

**产品调用**：显式传 `cross_natal_path=<...>` + 触发条件参数。

**默认策略**：用户没提上述任一 → **不启用**，本命交叉段不出现，判读走纯 Prashna。

**混叠限制**：本命交叉可与 Tajika 副层共存（都是 Parashari 生态内），可与 KP 独立栈共存（本命交叉的 Dasha 层与 KP 不打架）；但**未来若新增交叉判据要与 KP 混用，需在本 policy 里明写规则并加回归测**。

## 3. V1 判据 · 矛盾裁决

### 3.1 判据流程

1. 主判读 Parashari 走完 5 步，得三档结论 + 时间窗
2. 读本命 Vimshottari Dasha 当前段：Dasha_主 / Antar_主
3. 读本命相关宫（问题对应宫）主星状态：落宫、尊贵度、SAV
4. 交叉检查：

| 检查项 | 一致（判读单标注"本命层与 Prashna 同向"） | 打架（本命裁决） |
|---|---|---|
| **方向 A**：Prashna 判"成" | 本命当前 Dasha 主 / 相关宫主 落 Kendra/Trikona 或有 Yoga → 一致 | 本命当前 Dasha 主落 6/8/12 且无救援 → 打架，Prashna 降"悬"或"不成" |
| **方向 B**：Prashna 时间窗在 X~Y 年内 | 本命 Dasha 段 X~Y 与相关宫主/事件 Karaka 有身份重合 → 一致 | 本命该段 Dasha 主是 6/8/12 主且落 dushtana → 打架，Prashna 时间窗改"悬"或推迟到下一段 |
| **方向 C**：Prashna 判"不成" | 本命当前 Dasha 主也落 dushtana → 一致，加强"不成" | 本命当前 Dasha 主为强 Yogakaraka 落 Kendra → 微弱打架，Prashna 结论保留但补"本命层反例，需重新审视问题澄清" |

### 3.2 裁决规则

- **打架 = 本命裁决**：Prashna 结论按本命方向调整档次（成→悬→不成 逐档降；反向不逆升）
- **一致 = Prashna 结论保留**：判读单标注"本命层与 Prashna 同向，结论稳固"
- **单向不逆升**：本命层永不把 Prashna "不成"升到"成"（本命裁决只用于**收紧**结论，不用于**扩张**）——保守原则
