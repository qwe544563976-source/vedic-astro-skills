# Prashna 问题分类树 + 澄清话术 + 提问地推断详表

> **用途**：把用户任意问法归到 `house-karaka-map.md` 里 20+ 类中的一类；处理模糊/多问/歧义问题；补前置门控里"提问地→时区/经纬"详表。
> **调用时序**：SKILL.md 前置门控 → 输入检测 → 走本文件澄清 → 归类 → 起盘。

---

## 1. 问题分类树（快速定类）

按问题的**动词+对象**归类：

```
问 "能不能" / "会不会" / "是不是"
├─ 关于人/关系 → 情感/婚姻 或 人际/合作
│   ├─ 伴侣/婚姻 → 结婚能否 / 恋爱能否 / 复合 / 分手
│   ├─ 家人 → 父母健康 / 子女前途 / 家庭矛盾
│   └─ 商业 → 合作能否 / 协商谈判
├─ 关于事业/工作 → offer/换工作/升职/创业/辞职/被裁
├─ 关于钱 → 投资/买房/借贷/失物/大额消费
├─ 关于健康 → 康复 / 手术 / 慢性
├─ 关于学业/资格 → 考试 / 留学
├─ 关于地点/迁移 → 搬家 / 长途 / 移民
├─ 关于官非 → 官司胜负
└─ 关于修行 → 出家/皈依

问 "何时" / "什么时候"
→ 归到对应类，判读单在 Step 4 择时层重点展开

问 "选哪个"（AB 二选一）
→ 拆成两张盘：为 A 起一张问"接受 A 能否成"，为 B 起一张问"接受 B 能否成"
→ 或走"复合问题澄清模板"引用户自决只问一个
```

## 2. 澄清话术（用户提问不合规时）

### 2.1 模糊问题（"帮我看看运势""感情怎么样"）

**触发**：问句没有明确动作或时间指向。
**响应模板**：

> "Prashna 卜卦要一个**具体、能判定"是/否/何时"**的问题。'看看运势' 这样的问法我起盘也答不了。
> 你现在最想知道的是哪一件事？举几个可判定的例子：
> - 「我这份 offer 该不该接?」（能不能成）
> - 「我和 XX 今年能不能结婚?」（何时成）
> - 「我这个官司会赢吗?」（吉凶结论）
> 你选一个方向告诉我，我就起盘。"

### 2.2 多问题合并（"我想问事业和婚姻"）

**触发**：一句话里塞了 2 个及以上分类。
**响应模板**：

> "Prashna 一张盘只答一问——多问混判反而不清楚。
> 你现在想先问哪一个？另一个我们等下再起一张盘。"

**不建议合并起盘**：因为 Prashna 是"提问那一刻"的答案，两个问题的心念不重合，混判会让 Moon 权重错位。

### 2.3 假设性问题（"如果我 XX 会怎样"）

**触发**：句式带"如果/假设/万一"。
**响应模板**：

> "'如果……会怎样' 这种假设 Prashna 的判据不明确。我给你转成两种可判定的问法，你选一个：
> - 「我要不要做 XX?」（决策判据 → Lagna+Moon 是否指向 XX）
> - 「如果我做 XX，能不能成?」（结果判据 → 相关宫+Karaka 是否 支持）
> 选哪个?"

### 2.4 已发生事件（"上周面试怎么样"）

**触发**：问句时间已过 + 用户此刻起盘。
**判定**：**允许**——Prashna 可用于"复盘"，提问时刻依然是"我此刻问的这一刻"，Prashna 盘依然有效。
**注意**：如果用户复盘的是很久前的事（超过 1 年），提醒用户"提问时刻的盘反映的是**你此刻的心念**，而非事发时的状态——盘会告诉你事的**最终走向**，但不是当时的过程细节"。

### 2.5 反复起盘同一问题（拒绝）

**触发**：用户在 24 小时内就同一问题多次起盘 / 明说"我之前问过了，再看看"。
**响应模板**：

> "Prashna Marga 明训：'心念不定，盘不可信'。同一问题反复起盘会稀释判据，第二次起的盘无效。
> 上次判读是什么结论? 我们基于那张盘的证据展开讨论(走 Q&A 追问)，而不是重起。
> 如果距上次提问已 > 3 个月、或事已发生明显变化，可以起新盘。"

### 2.6 时间未定（"我以后能不能……"）

**触发**：完全无时间锚点。
**响应模板**：

> "我可以直接给"能不能 + 大致时间窗"。但如果你有一个大致的关心时段（比如"今年内""3 年内"），告诉我 → 择时判据会更聚焦。"

## 5. 分类不明确时的兜底

若走完本文档所有澄清后仍无法归到 `house-karaka-map.md` 里 20+ 类中的一类：

> "你这个问题超出 Prashna 的标准判据范围。Prashna 擅长回答'能不能/何时/是不是'的**离散判定题**；如果你要问的是'性格分析''命运趋势''长期人生方向'，这些应该走本命盘(vedic-core)，我可以帮你从那个入口进入。"

**不硬拗**——归不了类的问题直接告知用户走别的入口，比强行起盘给敷衍结论更负责。
